# AcidEngine — Architecture Overview

## Базовый принцип

Контракт является единственным источником истины. Все правила данных хранятся исключительно в Contract. Никакая другая сущность не должна содержать собственную копию правил.

## Основные компоненты

### Field
Контракт отдельного значения. Содержит:
- `type` — ожидаемый тип (int, str, float и т.д.)
- `min`, `max` — числовые границы
- `min_length`, `max_length` — границы длины строк
- `regex` — регулярное выражение
- `choices` — список допустимых значений
- `validators` — список пользовательских функций

Метод `validate(value)` проверяет значение и выбрасывает `ContractViolation` при ошибке.

### Contract
Центральная сущность системы. Описывает правила для набора данных:
- `schema` — словарь Field для структурированных данных
- `item_contract` — одиночный Field для простых значений
- `unique` — гарантия уникальности
- `ordered` — сохранение порядка вставки
- `frozen` — начальная заморозка контейнера
- `_rules` — список cross‑field правил

Методы:
- `create_container()` — создаёт Container
- `validate(data)` — проверяет данные и возвращает ValidationResult
- `add_rule(rule)` — добавляет cross‑field правило
- `from_yaml()`, `to_yaml()` — импорт/экспорт контракта
- `from_data()` — генерация контракта по образцу данных
- `generate_pipeline()` — генерация каркаса Python-пайплайна

### Container
Хранилище данных, которое соблюдает контракт. Поддерживает list, set, dict. Ключевой метод `add(item)`:
1. Проверяет, что контейнер не заморожен.
2. Применяет схему или item_contract.
3. Применяет cross‑field правила (если схема задана).
4. Проверяет тип, уникальность, валидаторы уровня контейнера.
5. Добавляет элемент в `_data`.

### QualityGate
Управление политикой обработки ошибок. Четыре режима:
- `strict` — полный откат при превышении порога ошибок
- `recovery` — откат с сохранением хороших записей
- `audit` — откат с сохранением всех записей
- `quarantine` — фильтрация без отката

Метод `add(item)` оборачивает `container.add()`, считает ошибки и принимает решение об откате. `ErrorRecord` собирает информацию о каждом нарушении.

### Explain Engine
`ValidationResult` содержит:
- `valid_data` — прошедшие валидацию записи
- `errors` — список `ErrorRecord`
- `summary()` — словарь со статистикой и топом нарушений
- `explain()` — человекочитаемый отчёт
- `to_markdown(filepath)` — экспорт отчёта в Markdown

### AI Guard
Модуль `ai.py` содержит класс `AIGuard` для валидации ответов LLM. Метод `validate(response_dict)` проверяет JSON-ответ нейросети через контракт. Метод `validate_json_string(json_string)` сначала парсит JSON, затем проверяет.

### Pipeline Generator
Метод `Contract.generate_pipeline(output_path)` генерирует Python-скрипт на основе `pipeline_meta` контракта. Ожидаемые ключи: `input`, `output`, `processing`. Генерируются функции-заглушки и оркестратор.

### LangChain Plugin
Класс `AcidOutputGuard` реализует совместимый с LangChain интерфейс для валидации выходных данных LLM. Принимает контракт и параметр `on_failure` (`"raise"` или `"ignore"`).

## Поток данных

1. Пользователь описывает Contract.
2. Contract создаёт Container.
3. Данные добавляются через Container.add() или QualityGate.add().
4. При ошибках генерируется ContractViolation с ожидаемым и полученным значениями.
5. QualityGate может откатить транзакцию.
6. ValidationResult предоставляет отчёт.
